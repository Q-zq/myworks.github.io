GitHub 部署指南

本指南将帮助您将 essays 目录部署到 GitHub Pages，实现自动更新。

步骤 1：获取 GitHub 个人访问令牌

  1. 登录您的 GitHub 账号：https://github.com
  2. 点击右上角的头像，选择「Settings」
  3. 在左侧菜单中选择「Developer settings」
  4. 选择「Personal access tokens」→「Tokens (classic)」
  5. 点击「Generate new token」按钮
  6. 填写 token 名称（例如：deploy-token）
  7. 在「Select scopes」中勾选「repo」权限
  8. 点击「Generate token」按钮
  9. 复制生成的 token（请妥善保存，token 只会显示一次）

步骤 2：配置部署脚本

  1. 打开 simple-deploy.js 文件
  2. 将第 6 行的 GITHUB_TOKEN 值替换为您刚才生成的 token
  3. 保存文件

步骤 3：运行部署脚本

  1. 在 essays 目录中打开终端
  2. 运行命令：node simple-deploy.js
  3. 等待部署完成

步骤 4：验证部署结果

  1. 访问您的 GitHub Pages 网站：https://q-zq.github.io/myworks.github.io/myworks/essays/
  2. 确认网站内容已更新

自动部署设置

要实现文件变化时自动部署，可以使用以下方法：

方法 1：手动运行脚本

每次修改文件后，手动运行 node simple-deploy.js 命令。

方法 2：使用 PowerShell 监控

  1. 创建一个名为 watch.ps1 的文件
  2. 将以下内容复制到文件中：
    3. $watcher = New-Object System.IO.FileSystemWatcher
$watcher.Path = "."
$watcher.Filter = "*.*"
$watcher.IncludeSubdirectories = $true
$watcher.EnableRaisingEvents = $true

$action = {
    $path = $Event.SourceEventArgs.FullPath
    $changeType = $Event.SourceEventArgs.ChangeType
    Write-Host "文件 $changeType: $path"
    Write-Host "触发部署..."
    node simple-deploy.js
}

Register-ObjectEvent $watcher "Changed" -Action $action
Register-ObjectEvent $watcher "Created" -Action $action
Register-ObjectEvent $watcher "Deleted" -Action $action

Write-Host "监控已启动，文件变化时将自动部署"
while ($true) { Start-Sleep 1 }
  3. 运行命令：powershell -ExecutionPolicy Bypass -File watch.ps1

注意事项

  • 确保 GitHub 令牌具有 repo 权限
  • 不要将令牌分享给他人
  • 对于大型文件，可能会遇到 GitHub API 的大小限制
  • 部署过程中会在终端显示详细的日志信息

部署成功

当您看到终端显示「部署完成！」时，说明文件已成功上传到 GitHub。
